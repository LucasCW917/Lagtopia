TCP Router Documentation

This TCP router allows multiple services to communicate via standardized commands. It is language-agnostic and works with Go, Python, or any language that implements the length-prefixed framing protocol.

Supported Commands

  • REGISTER <SERVICE_NAME> - Registers a service with the router.
  • UNREGISTER <SERVICE_NAME> - Unregisters a service.
  • SENDTO <SERVICE> "<payload>" - Sends a message to a specific service.
  • BROADCAST "<payload>" - Sends a message to all connected services.
  • PING - Router replies with PONG.
  • LIST - Returns all registered services.

Message Framing

All messages are sent as 4-byte big endian length prefix + message bytes.

+-----------------+---------------------+
| 4 bytes length  | message payload     |
+-----------------+---------------------+

Using From Go

conn, _ := net.Dial("tcp", "localhost:9000")
defer conn.Close()

writeMessage(conn, "REGISTER LOGIN")
writeMessage(conn, `SENDTO SERVER "login:TRUE"`)

Using From Python

import socket, struct

def send_message(sock, msg):
    data = msg.encode()
    length = struct.pack('>I', len(data))
    sock.sendall(length + data)

def recv_message(sock):
    length = struct.unpack('>I', sock.recv(4))[0]
    return sock.recv(length).decode()

sock = socket.create_connection(('localhost', 9000))
send_message(sock, "REGISTER LOGIN")
send_message(sock, 'SENDTO SERVER "login:TRUE"')
response = recv_message(sock)
print(response)

Notes

  • Router automatically removes services that disconnect.
  • All payloads are strings; parsing is up to the service.
  • Maximum message size: 1MB.
  • Errors are returned in the format: ERROR <message>
